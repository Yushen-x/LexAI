# LexAI 研发任务清单

## 一、项目战略与赛题对齐

- [ ] 明确最终主打赛题方向：智能体 / 工具应用 / 系统解决方案
- [ ] 确定项目主叙事：高可信法律智能体、法律工作台或智慧法律服务平台
- [ ] 明确 2-3 个最具展示力的核心场景
- [ ] 输出一句话定位、项目 Slogan 与答辩主标题
- [ ] 梳理比赛交付物清单、节点与负责人

## 二、产品定义与业务建模

- [ ] 明确目标用户画像：公众、律师、企业法务、法律援助机构
- [ ] 梳理典型任务流：咨询、检索、分析、生成、审查、整理
- [ ] 设计系统信息架构与主交互流程
- [ ] 明确每个场景的输入、输出与评价标准
- [ ] 设计高风险问题的保守回答与人工复核机制

## 三、总体技术架构设计

- [ ] 确定腾讯系 AI 工具平台接入方案与调用模式
- [ ] 设计系统分层架构：接入层、编排层、模型层、检索层、治理层
- [ ] 设计多智能体协同流程与任务路由机制
- [ ] 设计结构化输出规范与模板体系
- [ ] 设计日志、权限、审计、异常处理与可观测性方案

## 四、法律知识工程建设

- [ ] 收集法律法规、司法解释、典型案例、合同模板、文书模板
- [ ] 设计知识分类体系：法条、案例、模板、问答、规则
- [ ] 设计知识切分、标签化与索引策略
- [ ] 建立可检索知识库与版本管理机制
- [ ] 建立示例问答集、任务集与评测样本集

## 五、核心智能能力研发

### 1. 法律任务识别与路由

- [ ] 实现咨询、分析、生成、审查、整理等任务类型识别
- [ ] 实现法律子领域识别，如劳动、婚姻、合同、侵权、合规
- [ ] 实现基于任务类型的工作流分发

### 2. 法律知识增强检索

- [ ] 实现关键词检索与语义检索结合的混合召回
- [ ] 实现法律依据、案例、模板的分通道检索
- [ ] 实现检索结果重排与来源标注
- [ ] 实现引用片段回传与结果可追溯展示

### 3. 多智能体协同编排

- [ ] 设计咨询分诊智能体
- [ ] 设计法律检索智能体
- [ ] 设计争议分析智能体
- [ ] 设计文书生成智能体
- [ ] 设计结果审校智能体

### 4. 法律文本理解与信息抽取

- [ ] 实现主体识别、事实提取、时间线梳理
- [ ] 实现争议焦点提炼与证据缺口识别
- [ ] 实现合同条款切分、标签化与风险点定位
- [ ] 实现长文本层级摘要与多材料关联分析

### 5. 结构化文书与报告生成

- [ ] 设计起诉状、答辩状、律师函、合同审查报告模板
- [ ] 实现模板约束下的结构化生成
- [ ] 实现生成结果字段校验、缺失提醒与格式规范化
- [ ] 实现结果导出与二次编辑能力

## 六、可信 AI 与安全治理

- [ ] 建立法律回答风险分级机制
- [ ] 建立引用校验与依据绑定机制
- [ ] 建立高风险结论的保守回答策略
- [ ] 建立免责声明与人工复核提示机制
- [ ] 建立敏感信息脱敏、权限控制与日志审计机制

## 七、前后端系统实现

- [ ] 完成前端高保真页面设计与主流程开发
- [ ] 完成用户输入、文件上传、任务执行、结果展示页面
- [ ] 完成后端接口、任务编排、状态管理与结果存储
- [ ] 完成知识库接入、历史记录与操作日志功能
- [ ] 完成多角色工作台或主控面板设计

## 八、效果评测与性能优化

- [ ] 建立法律咨询评测集
- [ ] 建立案件分析与合同审查评测集
- [ ] 建立文书生成结构完整性评测指标
- [ ] 统计引用可追溯率、风险识别覆盖率、输出稳定性
- [ ] 优化响应速度、流程成功率与展示效果

## 九、比赛演示与材料包装

- [ ] 设计最具冲击力的 Demo 主流程
- [ ] 准备标准演示数据、案例文本与合同样本
- [ ] 梳理技术亮点、创新点、难点与突破点
- [ ] 输出答辩 PPT 文案与项目说明材料
- [ ] 准备演示视频、讲解脚本与答辩问答预案

## 十、建议分工

- [ ] 产品与方案负责人：负责场景定义、材料整合、答辩主叙事
- [ ] AI 与算法负责人：负责模型接入、知识增强、Prompt、评测优化
- [ ] 后端负责人：负责任务编排、接口、数据流转、日志与权限
- [ ] 前端负责人：负责界面交互、结果展示、Demo 体验与可视化表达
- [ ] 测试与交付负责人：负责评测、演示排练、材料输出与交付验收

## 十一、里程碑规划

- [ ] 里程碑 1：完成项目定位、系统架构、原型设计与主叙事确定
- [ ] 里程碑 2：完成法律知识库、核心工作流与最小可用 Demo
- [ ] 里程碑 3：完成多智能体协同、结构化输出与可信治理能力
- [ ] 里程碑 4：完成评测优化、视觉包装与比赛交付材料

## 十二、答辩重点准备

- [ ] 为什么法律场景需要专用 AI 系统，而不是通用聊天机器人
- [ ] 项目如何体现技术深度，而不是简单套壳
- [ ] 项目如何控制模型幻觉与法律风险
- [ ] 项目如何体现社会价值、产业价值与落地潜力
- [ ] 项目后续如何扩展至法律援助、企业合规与司法辅助场景
