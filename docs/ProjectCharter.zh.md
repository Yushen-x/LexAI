## 3. 项目描述

### 3.1 背景

LexAI 是一个学校课程项目，旨在在可控原型场景下探索 AI 对法律服务流程的辅助作用。项目围绕当前法律服务需求增长、优质资源分布不均，以及大语言模型、检索增强和结构化输出方法带来的新机会展开。

### 3.2 挑战或机遇描述

| 项目 | 描述 |
| --- | --- |
| 挑战 1 | 专业法律服务在不同地区和用户群之间仍然分布不均。 |
| 挑战 2 | 个人和小型组织在咨询与文书准备方面通常面临较高成本。 |
| 挑战 3 | 法律工作中存在大量重复性的检索、起草和审查任务，适合引入 AI 辅助。 |
| 机遇 | 课程项目原型可以先验证 AI 辅助法律工作流的可行性，再决定后续是否继续扩展。 |

### 3.3 期望影响概览

| 影响领域 | 期望影响 |
| --- | --- |
| 效率 | 缩短代表性法律咨询和文书起草任务的处理时间。 |
| 质量 | 提升原型输出的一致性和可追溯性。 |
| 展示 | 形成一个可用于校内展示的小范围原型，覆盖法律咨询、案件分析、合同审查和部分文书生成流程。 |
| 延展性 | 为后续迭代沉淀可复用的技术和评测基础。 |

## 5. 项目范围

### 5.1 本项目将包含的内容

| 范围内事项 | 描述 |
| --- | --- |
| 法律咨询 | 面向常见法律问题的原型问答流程，输出结构化回答和边界提示 |
| 案件分析 | 针对选定场景提供基础案件摘要与问题分析支持 |
| 合同审查 | 对样本合同条款进行风险识别并给出修改建议 |
| 文书生成 | 针对有限模板提供部分法律文书起草支持 |
| 知识支持 | 围绕试点场景构建基础法律知识整理与检索能力 |
| 系统交付 | 完成内部演示部署、小范围试点测试、日志记录和评测留痕 |

### 5.2 本项目范围外的内容

| 范围外事项 | 描述 |
| --- | --- |
| 正式法律代理 | 不提供法院、仲裁或纠纷处理中的正式代理服务 |
| 具有约束力的法律意见 | 未经专业审核，系统输出不作为正式法律意见 |
| 商业化运营 | 不提供生产级公共服务或 24/7 商业化部署 |
| 全领域法律覆盖 | 不追求覆盖全部法律子领域或国际法律体系 |
| 完整企业级治理体系 | 不建设超出课程项目需要的企业级合规体系 |
