# 得理直连快速接入

当前项目已经调整为：

- 默认运行模式：`deli`
- 只要配置好得理开放平台的 `APP_ID` 和 `SECRET`
- 就可以先跑通“法律检索增强 + 规则兜底”的版本

这条路线适合当前比赛冲刺阶段先把系统跑通，不必被元器发布流程卡住。

## 当前行为

在 `deli` 模式下：

- 法律咨询：优先调用得理法规检索 + 类案检索，再生成结构化建议
- 案件分析：优先调用得理类案检索，再生成事实/焦点/证据缺口/建议
- 合同审查：优先结合得理法规检索，再走规则化风险审查
- 合同起草：使用本地模板和规则生成合同草案

说明：

- 这不是“只有搜索，没有回答”
- 而是“得理检索增强 + 系统内结构化兜底”
- 对 demo 来说已经足够跑主链路

## 你需要填写的配置

在 `backend/.env` 中至少填这两个：

```env
DELI_APP_ID=你的得理APP_ID
DELI_SECRET=你的得理SECRET
LEXAI_AI_MODE=deli
```

如果你还没创建 `backend/.env`，可以从：

- [backend/.env.example](D:\Desktop\LexAI\backend\.env.example:1)

复制一份。

## 可选配置

如果你还想保留本地知识库增强，也可以再填：

```env
LEXAI_KNOWLEDGE_PATH=
LEXAI_KNOWLEDGE_INDEX_PATH=
```

如果暂时不填，也不影响系统先跑起来。

## 启动方式

### Windows

```powershell
cd D:\Desktop\LexAI
.\start.ps1
```

### 或直接启动后端

```powershell
cd D:\Desktop\LexAI\backend
.\mvnw.cmd spring-boot:run
```

## 验证是否生效

启动后可以先访问：

- 后端健康检查：`http://localhost:8081/api/actuator/health`

然后测试：

- 法律咨询
- 案件分析
- 合同审查
- 合同起草

## 当前默认策略

如果存在：

- `DELI_APP_ID`
- `DELI_SECRET`

系统会优先按 `deli` 模式运行。

如果得理没配置、元器也没配置，启动脚本会退回 `mock`。

## 后续如果要切回元器

只需要改：

```env
LEXAI_AI_MODE=yuanqi
```

并补齐元器相关的 `YUANQI_*` 环境变量即可。
